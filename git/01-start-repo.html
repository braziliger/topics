01 - Starting a repository

First steps

There are a few different ways you might want to initialize a repository. There are three cases I can think of -

  1. You have not started coding, and want to create a blank repo to start adding code to
  2. You have pre-existing code that you want to add to a repo, which you have not created yet
  3. You want to start working on a pre-existing repository that has code in it

So we'll go over a few simple steps - first, how to create a repository on GitHub. You are probably thinking "I already know this"! But read it anyway.

Then, we'll go over those three cases, and how to manage each case.

  • Creating a repository
  • No code, no repo

Creating a repository

To create a repository, go to GitHub and log in. The first screen you see should look something like this:

Click on the "new" button and you'll be presented with a screen like this:

Let's go over these fields:

  • Repository name is obviously the name of your repo
  • Description is nice to fill out if you're creating a public repo; if it's private that's really up to you
  • Public/Private dictates if random people on GitHub will be able to locate & clone your code or not

Notice the line that says Skip this step if you're importing an existing repository. If you have pre-existing code that you're wanting to add to a new repo, don't touch anything below this line. Otherwise, if you're working clean (you have no code yet):

  • Add a README file will do what it says - add a README.md file to your repository. More on this soon
  • Add .gitignore will add a special file called .gitignore to your repository. In this file, you can add paths to files that you do not want Git to track. More on this later on as well
  • Choose a license allows you to pick a license to add to your repo that tells people what they can/cannot do with your code; important if this is a public repo, and you can read up on what different licenses do on your own.
