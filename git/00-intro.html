What is Git?

Git is version control software. The specific job of Git is to track changes in a codebase. Here's a basic overview of the workflow of Git:

  1. Create a new repository, or find a pre-existing repository you want to work on
  2. "Clone" the repository to your local machine
  3. Make a change to the repository by adding, deleting, or modifying a file in the repository
  4. "Commit" the change you made to the repository
  5. Repeat steps 3 & 4 until you are ready to publish your changes
  6. "Push" the changes you made to the remote repository

There are obviously variations to this list but that's generally how it works. So, I'm building this site to walk you through how to use git. The next section will explain how to use Git in a very basic sense and will build up to more complex stuff.
